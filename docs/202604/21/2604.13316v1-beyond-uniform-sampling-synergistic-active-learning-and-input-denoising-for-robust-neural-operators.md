## 摘要
神经算子已成为物理模拟的快速替代模型，但它们仍然极易受到对抗性扰动的影响，这对于安全关键的数字孪生部署而言是严重隐患。本文提出了一种协同防御方法，将基于主动学习的数据生成与输入去噪架构相结合。主动学习部分通过差分进化攻击自适应地探测模型弱点，在发现的易损位置生成有针对性的训练数据，同时通过自适应平滑比率保护机制维持基线精度。输入去噪部分在算子架构中引入一个可学习的瓶颈结构，用于滤除对抗性噪声，同时保留与物理相关的特征。在粘性 Burgers 方程基准测试中，该组合方法实现了 2.04% 的综合误差（1.21% 基线 + 0.83% 鲁棒性），相较于标准训练（15.42% 综合误差）减少了 87%，并优于仅采用主动学习（3.42%）或仅采用输入去噪（5.22%）的情况。更广泛地说，我们的结果结合以往关于跨架构易损性分析的研究表明，神经算子的最佳训练数据具有架构依赖性：由于不同架构在不同输入子空间中集中敏感度，均匀采样无法充分覆盖所有模型的易损性分布。此发现对于神经算子在包括核反应堆监测等安全关键能源系统中的部署具有潜在意义。

## Abstract
Neural operators have emerged as fast surrogate models for physics simulations, yet they remain acutely vulnerable to adversarial perturbations, a critical liability for safety-critical digital twin deployments. We present a synergistic defense that combines active learning-based data generation with an input denoising architecture. The active learning component adaptively probes model weaknesses using differential evolution attacks, then generates targeted training data at discovered vulnerability locations while an adaptive smooth-ratio safeguard preserves baseline accuracy. The input denoising component augments the operator architecture with a learnable bottleneck that filters adversarial noise while retaining physics-relevant features. On the viscous Burgers' equation benchmark, the combined approach achieves a 2.04% combined error (1.21% baseline + 0.83% robustness), representing an 87% reduction relative to standard training (15.42% combined) and outperforming both active learning alone (3.42%) and input denoising alone (5.22%). More broadly, our results, combined with cross-architecture vulnerability analysis from prior work, suggest that optimal training data for neural operators is architecture-dependent: because different architectures concentrate sensitivity in distinct input subspaces, uniform sampling cannot adequately cover the vulnerability landscape of all models. These findings have potential implications for the deployment of neural operators in safety-critical energy systems including nuclear reactor monitoring.

## 一、研究背景与问题动机

- **研究核心问题**：  
  神经算子（Neural Operators）作为求解偏微分方程（PDE）的高效替代模型，被广泛用于数字孪生与能量系统仿真。然而，近年研究（Roy et al., 2026）发现其**极易受到对抗扰动攻击**：即即便输入仅发生物理上微小的变化，输出预测也可能剧烈偏离真实物理解。这种“偏离物理”的脆弱性对**安全关键系统（如核反应堆监测）**构成严重威胁。
- **根源分析**：  
  不同架构的神经算子在输入空间有不同的敏感度分布（通过雅可比矩阵谱得出的有效扰动维度 \(d_{eff}\) 量化），导致**均匀采样**的训练数据不能覆盖模型的所有潜在脆弱区域。
- **研究目标**：  
  提出一种可系统提升神经算子鲁棒性的防御机制，使其在物理一致性与抗攻击性间取得平衡。

## 二、方法论概述

### 1. 核心思想
作者提出了一个**协同防御框架**（见图 2），融合了：
- **主动学习（Active Learning）**：针对模型脆弱区域生成有针对性的训练数据。
- **输入去噪（Input Denoising）**：在网络结构中引入可学习压缩层（autoencoder bottleneck），过滤掉高频噪声。

这两者从 *数据层面* 与 *架构层面* 分别提升鲁棒性，协同作用显著优于二者独立使用。

---

### 2. 主动学习模块

- **攻击探测机制**：  
  使用**差分进化（Differential Evolution, DE）攻击**搜索模型弱点。
- **核心算法（Algorithm 1）**：  
  主动学习迭代流程包括以下步骤：
  1. 初始化若干平滑样本（baseline 数据集）。
  2. 训练当前模型并计算“基线误差”。
  3. 用 DE 攻击验证集以定位脆弱输入位置。
  4. 根据基线误差动态调整“平滑比率”参数 \( \alpha \in [0.3, 0.7] \)：  
     - 若精度下降，则提高平滑样本比例；
     - 若鲁棒性不足，则生成更多针对弱点的对抗样本。
  5. 合成物理校正数据（使用 PDE 求解器求得真实标签）。
  6. 重训练模型。
  7. 循环直至达到设定仿真预算 \(B\)。

- **与传统对抗训练相比**：  
  作者不使用“原始标签”对抗训练，而是**基于物理求解器的真实响应重新标注**，以确保训练信号符合物理规律，避免经典对抗训练中“精度–鲁棒性冲突”的问题。

---

### 3. 输入去噪模块

- **结构设计**：  
  在 DeepONet 的分支网络（branch network）前增加一个小型自编码器：
  \[
  \tilde{u}_0 = w \cdot D(u_0) + (1 - w) \cdot u_0
  \]
  其中：
  - \(D(\cdot)\) 是压缩–重建过程（64→32→64 维瓶颈结构，Tanh 激活）；
  - \(w = \text{sigmoid}(\omega)\) 为可学习融合权重，用于平衡“信息保留”与“噪声抑制”。
- **作用**：  
  降低有效输入维度，从而过滤扰动在高频/局部空间的放大效应。

---

### 4. 协同机制

- **主动学习** 侧重**针对性地学习易受攻击区域的真实解映射**；
- **去噪结构** 则提供**全局输入降噪与维度收缩**，即使遇到未被探测的扰动也能抗干扰；
- 二者结合构成“数据驱动 + 架构驱动”的鲁棒协同防御。

## 三、实验设计

### 1. 基准问题
- 使用 **一维粘性 Burgers 方程** 作为测试 PDE：
  \[
  \frac{\partial u}{\partial t} + u \frac{\partial u}{\partial x} = \nu \frac{\partial^2 u}{\partial x^2}, \quad \nu = 0.1, \, T=1
  \]
- 输入为初始条件 \(u_0(x)\)，输出为终态 \(u(x,T)\)。

### 2. 数据生成
- 采用**谱方法求解器（spectral solver）**生成 “真解” 数据；
- 总仿真预算：**600 次数值求解**；
  - 初始样本：50；
  - 每轮主动学习新增：20；
  - 约 10–15 次迭代完成预算。

### 3. 攻击与评估
- 攻击方式：差分进化攻击（30 次迭代用于探测，40 次用于评估）。
- 指标：
  - **Baseline Error**：干净数据集上的 \(L_2\) 误差；
  - **Robustness Error**：被攻击样本下预测解与物理解的偏差；
  - **Combined Score** = 二者之和，衡量准确–鲁棒性的综合性能。

### 4. 对比方法
| 策略 | 数据策略 | 模型结构 | 说明 |
|------|------------|-----------|--------|
| Baseline | 平滑采样 | 标准 DeepONet | 无防御 |
| Balanced | 混合采样（60%扰动 + 40%平滑） | 标准结构 | 随机增强 |
| Denoising only | 平滑采样 | 含去噪模块 | 架构防御 |
| Active Learning (AL) | 自适应采样 | 标准结构 | 数据防御 |
| AL + Denoising | 自适应采样 | 含去噪模块 | 协同方案 |

## 六、主要结论与发现

1. **协同防御显著提升鲁棒性**：  
   - 组合方法（AL + Denoising）总误差 2.04%，比标准训练（15.42%）降低 87%。  
2. **数据与架构互补性**：

4. **实证意义**：  
   - 单纯验证集精度并不代表真实鲁棒性；
   - 数据–结构协同策略对安全关键工业系统尤为必要。
