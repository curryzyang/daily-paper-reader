## 摘要
利用网络交通流模型进行优化需要计算目标函数对模型参数的梯度。传统方法依赖数值微分或无导数方法，这些方法在参数维度较高时扩展性较差；或依赖伴随方法，这类方法需要对每个特定模型手工推导。本研究提出了一种基于链路传输模型（Link Transmission Model, LTM）的端到端可微分网络交通流模拟器，结合了一般节点模型和动态用户最优（Dynamic User Optimum, DUO）路径选择模型。我们发现，LTM通过分段线性的 $\min$/$\max$ 运算作用于连续的聚合状态变量（累计车辆计数），这些运算几乎在所有点处都有次梯度，因此不需要平滑放松即可进行自动微分（Automatic Differentiation, AD）。我们将DUO路径选择模型及其Logit扩展纳入其中，以在保持可微性的同时显式考虑出行者的内生动态路径选择，因为分岔比是关于各目的地车辆计数的连续函数。最终得到的模拟器可在单次反向传播中通过反向模式自动微分精确计算梯度，而不依赖参数维度。为了验证所提模型的能力，我们在Chicago-Sketch数据集（约2500条链路和100万辆车，含15000个决策变量）上求解了一个动态拥堵收费优化问题。所提出的模型成功在约2小时内通过10000次迭代得到高质量解，即一次模拟与梯度计算耗时约0.8秒。该模拟器使用Python和JAX实现，并作为开源软件发布，名为UNsim（https://github.com/toruseo/UNsim）。

## Abstract
Optimization using network traffic flow models require computing gradients of objective functions with respect to model parameters. Conventional approaches rely on numerical differentiation or derivative-free methods that does not scale well with the parameter dimension, or on adjoint methods that require manual derivation for each specific model. This study proposes a novel end-to-end differentiable network traffic flow simulator based on the Link Transmission Model (LTM), incorporating general node models and a dynamic user optimum (DUO) route choice model. We observe that the LTM operates on continuous aggregate state variables (cumulative vehicle counts) through piecewise-linear $\min$/$\max$ operations, which admit subgradients almost everywhere and thus require no smooth relaxation for automatic differentiation (AD). We incorporate the DUO route choice model and its logit extension to explicitly consider endogenous dynamic route choice of travelers while preserving differentiability, because the diverge ratios are continuous functions of per-destination vehicle counts. The resulting simulator computes exact gradients via reverse-mode AD in a single backward pass regardless of the parameter dimension. In order to demonstrate the capability of the proposed model, we solved a dynamic congestion toll optimization problem on Chicago-Sketch dataset with around 2500 links and 1 million vehicles with 15 000 decision variables. The proposed model successfully derived a high quality solution with 10 000 iterations that took about 2 hours, meaning that 1 simulation run and gradient derivation took 0.8 second. The simulator, implemented in Python and JAX, is released as an open-source software named UNsim (https://github.com/toruseo/UNsim).

## 二、方法论与技术细节

### 1. 核心思想

- **模型基础**：基于 **Link Transmission Model (LTM)** 的一阶宏观交通流模型。该模型描述车辆累计计数在链路上的时空传输过程，是 LWR 理论的离散化形式。
- **关键创新**：  
  将所有流程（需求–供给计算、节点流分配、路径选择）表述为**连续且分段线性运算（min/max）**，这些运算在几乎所有点上都有次梯度，天然适用于自动微分。

### 2. 组成结构

1. **链路模型（LTM）**：  
   - 基于三角形基本图（Triangular Fundamental Diagram），定义自由流速度、交通容量、拥堵波速和拥堵密度。  
   - 用累计车辆计数 \(N_{l,U}(t)\)、\(N_{l,D}(t)\) 表示链路两端流量状态。
   - 通过分段函数计算需求与供给，并用 min/max 保持可微性。

2. **节点模型**：  
   含六类节点：
   - 起点/终点节点（处理外部需求）  
   - 空节点（dummy）  
   - 合流/分流节点（merge/diverge）  
   - 一般多入口多出口节点，采用 **Incremental Node Model (INM)**，将需求和供给约束转化为分步流量分配问题。  
   - 此模型在实现上用固定步数的循环，所有逻辑通过 min/max 表达而避免离散条件语句，保证可微分。

3. **路径选择模型（DUO & Logit-DUO）**：
   - 基于动态用户最优（Dynamic User Optimum, DUO）原则：每个时刻车辆选择最短瞬时路径。
   - 通过时间依赖最短路径算法（Bellman–Ford）确定流向；分流比为各目的地车辆计数的连续函数。
   - 若存在零梯度问题，引入 **Logit-DUO** 模型，用软化的概率函数平滑路径选择，提高梯度连续性。

4. **自动微分实现**：
   - 利用 Python + JAX 框架的 `jax.grad`, `jax.lax.scan` 实现反向与前向模式 AD。
   - 所有仿真步骤封装为函数链 `x_{t+1} = g(x_t, t; θ)`，梯度通过全时序反向传播自动计算。

5. **总体算法结构**：
   - 初始化状态 → 时间步扫描（Scan）→ 累计计数更新 → 目标函数计算 → 反向传播计算梯度。  
   - 与伴随方法数学上等价，但无需人工推导。

- **决策变量**：每条链路在40个时间段的收费，共 15,320 个参数。  
- **对比基准**：展示与无收费方案的性能对比，无外部基线模型。
